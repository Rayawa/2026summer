## 三、项目概述

### 3.1 项目选题：石头剪刀布手势识别（Rock-Paper-Scissors Detection）

本项目基于 YOLO11n 目标检测模型，训练一个能够实时识别摄像头画面中**剪刀（Scissors）、石头（Rock）、布（Paper）**三种手势的检测器。

**项目意义**：
- 手势识别是人机交互的重要技术方向，广泛应用于虚拟现实、智能家居、手语识别等领域。
- 通过本项目可以系统掌握深度学习目标检测的完整流程。

### 3.2 数据集介绍

数据集来自 Roboflow 上的 [rock-paper-scissors-sxsw](https://universe.roboflow.com/roboflow-58fyf/rock-paper-scissors-sxsw) 公开数据集（v11 版），由 SXSW 2023《World's Largest Game of Rock, Paper, Scissors》挑战赛贡献者制作。

**数据集统计**：

| 数据集 | 图片数量 | 用途 |
|--------|----------|------|
| 训练集 | 14,966 | 模型训练 |
| 验证集 | 588 | 验证调参 |
| 测试集 | 320 | 最终评估 |
| **合计** | **15,874** | — |

**标注类别**：

| 类别 | 说明 |
|------|------|
| Paper | 布（手掌展开） |
| Rock | 石头（握拳） |
| Scissors | 剪刀（食指和中指伸出） |

#### 数据预处理

- 自动旋转：基于 EXIF 方向信息校正图像方向。
- 尺寸调整：统一缩放到 **640×640** 像素（拉伸填充）。
- 标签格式：YOLOv11 格式（归一化的边界框坐标）。

#### 数据增强策略

为提升模型泛化能力，每张原图通过以下增强生成 **7 个版本**：

| 增强方式 | 参数 |
|----------|------|
| 水平翻转 | 50% 概率 |
| 随机旋转 | -12° ~ +12° |
| 随机剪切 | -2° ~ +2°（水平+垂直） |
| 亮度调整 | -25% ~ +25% |
| 曝光调整 | -25% ~ +25% |
| 高斯模糊 | 0 ~ 1.5 像素 |
| 椒盐噪声 | 1% 像素 |

## 五、模型设计与构建

### 5.1 模型选型：YOLO11n

**YOLO（You Only Look Once）** 是一种实时目标检测算法，将目标检测视为回归问题，一次性预测图像中所有目标的类别和位置。

选择 **YOLO11n** 的理由：
- **轻量化**：n（nano）版本仅 258 万参数，模型体积约 5.5MB，适合实时推理。
- **高性能**：在保持轻量的同时，提供优秀的检测精度。
- **端到端训练**：从数据预处理到模型推理的全流程均被封装在 Ultralytics 框架中，开发效率高。
- **硬件友好**：能在消费级 GPU 上快速训练，甚至可以在 CPU 上进行推理。

**YOLO11n 模型结构**：

| 组件 | 层数 | 说明 |
|------|------|------|
| Backbone | — | 卷积神经网络骨干，提取图像特征 |
| Neck | — | 特征金字塔网络（FPN），多尺度特征融合 |
| Head | — | 检测头，输出边界框和类别概率 |
| **总计** | 101层 | 融合后的推理网络（含 258 万参数） |

### 5.2 损失函数与优化器

- **损失函数**：YOLO 内置的组合损失，包括分类损失（BCE Loss）、定位损失（CIoU Loss）和置信度损失。
- **优化器**：AdamW（自适应矩估计 + 权重衰减）
- **学习率**：Ultralytics 默认自适应学习率调度策略

## 六、模型训练与调优

### 6.1 训练配置

```python
from ultralytics import YOLO

model = YOLO("yolo11n.pt")  # 加载预训练权重

model.train(
    data="rock-paper-scissors-11/data.yaml",
    epochs=50,          # 训练轮数
    imgsz=640,          # 输入图片尺寸
    device="cuda",      # GPU 加速
    batch=16,           # 批大小
    workers=8,          # 数据加载线程数
    optimizer="AdamW"   # 优化器
)
```

**训练参数说明**：

| 参数 | 值 | 说明 |
|------|-----|------|
| model | yolo11n.pt | 基于 COCO 预训练权重的迁移学习 |
| epochs | 50 | 完整训练轮数 |
| imgsz | 640 | 输入图像尺寸 |
| batch | 16 | 每批处理16张图片 |
| workers | 8 | 并行加载数据的进程数 |
| optimizer | AdamW | 带权重衰减的 Adam 优化器 |

### 6.2 训练过程

- **训练耗时**：50 epochs 共计约 **3.07 小时**
- **训练设备**：NVIDIA GeForce RTX 5060 Laptop GPU（8GB VRAM）
- **预训练初始化**：使用 `yolo11n.pt`（在 COCO 数据集上预训练）进行迁移学习，显著加速收敛。
- **自动验证**：每个 epoch 结束后自动在验证集上评估，保存最佳权重 `best.pt`。

## 七、模型评估与测试

### 7.1 验证集评估结果

使用 `evaluate.py` 在验证集（588张图片）上评估：

| 类别 | 图片数 | 实例数 | 精确率(P) | 召回率(R) | mAP50 | mAP50-95 |

**推理速度**：单张图片仅需 **2.1ms** 推理时间（GPU加速）。

### 7.2 测试集评估结果

使用 `test.py` 在测试集（320张图片）上评估：

| 类别 | 图片数 | 实例数 | 精确率(P) | 召回率(R) | mAP50 | mAP50-95 |
|------|--------|--------|-----------|-----------|-------|----------|
| **全部** | 320 | 210 | **0.898** | **0.920** | **0.934** | **0.693** |
| Paper | 72 | 72 | 0.868 | 0.861 | 0.904 | 0.630 |
| Rock | 63 | 72 | 0.938 | 0.944 | 0.951 | 0.715 |
| Scissors | 64 | 66 | 0.887 | 0.953 | 0.947 | 0.734 |

### 7.3 评估指标分析

- **mAP50 = 0.940（验证集）/ 0.934（测试集）**：在 IoU 阈值为 0.5 的条件下，平均精度高达 94%，表明模型能够准确检测手势。
- **mAP50-95 = 0.697 / 0.693**：严格评估指标（IoU 从 0.5 到 0.95 取平均），表现良好。
- **精确率与召回率**：模型在精确率和召回率之间取得了良好的平衡。
  - 验证集：P=0.942, R=0.891
  - 测试集：P=0.898, R=0.920
- **类别均衡性**：三个手势类别的检测性能接近，说明数据集标注均衡，模型没有对特定类别的偏向。

### 9.1 主要成果

1. 成功构建了基于 YOLO11n 的石头剪刀布手势检测系统。
2. 使用 Roboflow 公开数据集（15,874 张图片）完成了模型训练。
3. 验证集 mAP50 达到 **0.940**，测试集 mAP50 达到 **0.934**，均表现优异。
4. 实现了单张图片推理和实时摄像头检测两种应用模式。
5. 完整掌握了从数据下载、模型训练、评估优化到部署应用的深度学习工作流。

### 9.2 遇到的问题及解决方案

| 问题 | 解决方案 |
|------|----------|
| 数据集需科学上网下载 | 提前在国内网络环境下载好数据集，或使用代理 |
| GPU 显存有限（8GB） | 选择 YOLO11n 轻量模型，batch=16 适配显存 |
| 高置信度下漏检某些手势 | 将置信度阈值调整为 0.25~0.30，平衡精确率和召回率 |
| NMS 耗时警告 | 调整 NMS 参数，使用 Ultralytics 默认优化配置 |
| 实时检测窗口中文标题乱码 | 使用英文标题（"Rock/Paper/Scissors Detection"） |

### 9.3 收获与体会

通过本次深度学习项目，我获得了以下宝贵经验：

1. **迁移学习的价值**：使用 COCO 预训练权重初始化模型，大幅缩短了训练时间，仅用 50 epochs 就达到了高精度。这让我体会到迁移学习在深度学习实践中的重要性。

2. **数据增强的重要性**：数据集使用了丰富的增强策略（旋转、翻转、噪声、模糊等），有效提升了模型的泛化能力，使模型在各种光线和角度下都能准确识别手势。

3. **模型轻量化的意义**：YOLO11n 仅 258 万参数、5.5MB 模型体积，在消费级 GPU 上可实现毫秒级推理，充分体现了"小模型大能力"的设计理念。

4. **端到端工具链的便利**：Ultralytics 框架封装了从训练到部署的全流程，让我可以专注于算法设计和实验调优，而不是底层实现细节。

5. **软硬件协同的重要性**：了解了模型参数量、输入尺寸、批大小等因素与 GPU 显存的匹配关系，以及数据加载并行度对训练速度的影响。

### 9.4 改进建议

- **模型升级**：尝试 YOLO11s/m/l 等更大版本，或使用 YOLOv8/YOLOv9 等更新架构，可能进一步提升精度。
- **数据增强优化**：针对特定困难场景（暗光、遮挡、手部姿势极端）增加定向数据增强。
- **模型量化**：使用 INT8 量化将模型进一步压缩到 ~2MB，实现移动端部署。
- **多角度检测增强**：增加不同视角（俯拍、侧拍）的训练数据，提高实用性。
- **集成手势序列识别**：在检测的基础上加入时序分析，实现动态手势识别。
